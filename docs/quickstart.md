# Quickstart

## 1. Create a Convex project

```bash
npx convex dev
# Follow the prompts to create a project
```

This gives you a deployment URL like `https://happy-animal-123.convex.cloud`.

## 2. Copy the Convex functions

Copy the entire `convex/` directory into your project:

```
your-project/
├── convex/
│   ├── schema.ts
│   ├── events.ts
│   ├── stats.ts
│   ├── http.ts
│   ├── dashboard.ts
│   ├── crons.ts
│   └── _generated/   (auto-generated by npx convex dev)
├── src/
│   └── app.tsx
└── package.json
```

Run `npx convex dev` to deploy and generate types.

## 3. Install the SDK

```bash
npm install @codersoft/wisp
```

Initialize it as early as possible in your app:

```typescript
import { init } from "@codersoft/wisp";

init({
  convexUrl: "https://happy-animal-123.convex.cloud",
});
```

That's it. The SDK automatically:
- Creates a stable machine ID (localStorage + cookie fallback)
- Starts a 30-minute session window
- Fires a `session_start` event with browser metadata (UA, platform, screen, referrer)
- Tracks `pageview` on navigation (SPA or MPA)
- Tracks `window.onerror` and `unhandledrejection`
- Tracks clicks on elements with `[data-wisp-id]`
- Batches and flushes events every 5 seconds or 20 events, whichever comes first
- Flushes on tab close via `sendBeacon`

## 4. Open the dashboard

```bash
cd dashboard
npm install
npm run dev
# Opens at http://localhost:5173
```

Set `VITE_CONVEX_URL` in `dashboard/.env.local` to your deployment URL.

## 5. Verify

Send a manual event from the browser console:

```js
import wisp from "@codersoft/wisp";
wisp.track("test_event", { message: "hello!" });
```

It appears in the **Sessions** table within seconds.
